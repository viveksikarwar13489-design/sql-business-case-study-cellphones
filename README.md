# 📊 SQL Advanced Case Study – Cellphone Sales Analysis

## 🚀 Project Overview
This project is an advanced SQL case study based on a real-world business scenario involving cellphone sales data. The objective is to analyze transactional data and extract meaningful business insights using SQL.

The database simulates a **Cellphone Sales System**, including customers, products, manufacturers, and locations.

---

## 🗂️ Database Schema

The project is built using the following tables:

- **DIM_MANUFACTURER** – Manufacturer details  
- **DIM_MODEL** – Mobile model details  
- **DIM_CUSTOMER** – Customer information  
- **DIM_LOCATION** – Geographic data (State, Zip, Country)  
- **FACT_TRANSACTIONS** – Sales transaction data  

---

## 🎯 Key Objectives

This case study focuses on solving real business problems such as:

- Identifying sales trends across years  
- Finding top-performing manufacturers and models  
- Customer spending behavior analysis  
- Year-over-year growth tracking  
- Regional sales insights  

---

## 🧠 Skills & Concepts Used

- ✅ SQL Joins (INNER JOIN, etc.)
- ✅ Aggregations (SUM, AVG, COUNT)
- ✅ Window Functions (RANK, DENSE_RANK, LAG)
- ✅ CTEs (Common Table Expressions)
- ✅ Subqueries
- ✅ Case-based filtering
- ✅ Business problem solving

---

## 📌 Key Business Questions Solved

1. States with customers purchasing since 2005  
2. Top state buying Samsung phones  
3. Transactions per model per location  
4. Cheapest cellphone  
5. Avg price of models in top 5 manufacturers  
6. Customers with avg spend > 500 in 2009  
7. Models consistently in top 5 (2008–2010)  
8. 2nd top manufacturer (2009 & 2010)  
9. Manufacturers active in 2010 but not 2009  
10. Top 100 customers with yearly spend & growth %

---

## 📈 Sample Insights

- Identified consistent top-performing models across multiple years  
- Tracked customer spending growth using window functions  
- Found regional trends in cellphone sales  
- Compared manufacturer performance year-over-year
